1. Executive Summary

HANDOFF의 Phase 5 (Sync Engine)가 완료됐습니다. 5. ConflictResolver 결정 트리

표준 범위 (사용자 선택):

Conflict detected for path P │ ├── SHA256(local) == SHA256(remote)? │ └── YES → .identical, no archive, done. │ ├── |localMtime − remoteMtime| < 1s AND strategy != .mergeJSON? │ └── YES → .bothPreserved │ ├── write local copy as P.machine-A │ └── write remote copy as P.machine-B │ ├── strategy resolved by file extension: │ ├── .json/.yaml/.yml → .mergeJSON │ ├── .png/.jpg/.pdf/.zip/.mp4/.mov → .largerWins │ └── else → .newerWins │ ├── .mergeJSON: │ ├── parseJSON(local), parseJSON(remote) │ ├── deepMerge(local, remote, preferLocal: localNewer) │ │ └── on key conflict: newer side's value wins │ ├── on parse failure of either side → fall back to .newerWins │ └── write merged → both local AND remote, result .mergedInPlace │ ├── .largerWins: │ ├── localSize ≥ remoteSize → local wins │ ├── archive loser to ~/.claudesync/conflicts/YYYY-MM-DD/P.{role} │ └── overwrite loser side with winner │ └── .newerWins (default): ├── localMtime > remoteMtime → local wins ├── archive loser └── overwrite loser side with winner

archive는 30일 후 purgeOlderThan(days: 30)으로 자동 정리. 12개 테스트가 위 분기 모두 커버.

6. 실기 시연 절차

두 Mac 실기 검증은 docs/DEMO_TWO_MACS.md 참조. 3개 시나리오:

  • A. 메뉴바 + 온보딩 UI — 양 Mac에서 빌드 후 실행, 메뉴바 아이콘 + Open Onboarding → 5스텝 윈도우 확인.
  • B. xctest 자동 페어링 시연 — 한 Mac에서 EndToEndPairingTests + EndToEndSyncTests 실행. PairingManager 풀 핸드셰이크 (양측 6자리 코드 일치 + authorized_keys 라인 설치) + 실제 rsync로 파일 라운드트립 (create + delete).
  • C. 실 SSH로 두 Mac 사이 rsync 시연 — Mac A에서 SSH 키 생성, Mac B에 restrict,command="rsync --server …" 라인 설치, Mac A → Mac B로 비밀번호 없이 rsync 전송. restrict,command 가드 검증 (임의 ssh 명령 거부).
자동 검증 vs 수동 시연: 시나리오 B는 이번 세션에서 PASS 확인됨. 시나리오 C는 사용자의 두 Mac 환경이 필요해 가이드 형태로 출하 — 본 세션의 Claude Code는 한 Mac에서만 동작합니다.
