5. 디버깅 노트

Phase 4에서 마주친 비자명한 문제 3건과 해결 방법:

증상원인해결
FSEvents 콜백은 호출되는데 AsyncStream consumer가 0 events 수신 테스트 파일 생성 후 collectEvents를 호출해 iterator 설치 — 그 사이 yield는 enqueued 됐으나 for await가 시작되기 전에 task group의 deadline race로 cancel 패턴 변경: watcher.start() 직후 consumer Task를 먼저 spawn → 파일 변경 → 그 후 watcher.stop()으로 stream 종료. consumer.value를 await
Watched root /var/folders/.../tmp/...인데 FSEvents 이벤트 paths는 /private/var/folders/...로 도착 macOS의 /var → /private/var 심볼릭 링크. NSString.resolvingSymlinksInPath는 /var를 풀지 않음 realpath(3) C 함수를 직접 호출하여 canonical path 얻기
FileWatcherActor 테스트가 데드락 — xcodebuild가 영원히 안 끝남 actor.stopAll() 호출 후에도 outer outputContinuation이 finish() 안 됨 → consumer Task의 for await batch in stream이 영원히 대기 stopAll()에 outputContinuation.finish() 추가하여 lifecycle 정리
