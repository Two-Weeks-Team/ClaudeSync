핵심 설계 결정

  • xcodegen 채택: .pbxproj를 손으로 관리하지 않기 위함. 단, xcodegen이 Info.plist/entitlements를 재생성하지 않도록 info:/entitlements: 디렉티브를 제거하고 INFOPLIST_FILE / CODE_SIGN_ENTITLEMENTS 빌드 세팅으로만 연결.
  • NSLock 기반 LineBuffer: Swift 6 strict concurrency가 Pipe.readabilityHandler 클로저의 mutable var buffer = Data() 캡처를 거부. 클래스 + NSLock으로 Sendable 안전성 확보.
  • 로그 파일 위치 분리: 동기화 대상인 ~/.claude/가 아닌 ~/.claudesync/logs/를 사용. SyncTarget의 absolutelyExcluded 패턴과도 정합.

7. Phase 2 위험과 완화

위험심각도완화책
테스트가 실제 ~/.ssh/authorized_keys를 변조할 가능성 HIGH SSHKeyManager init에 homeDirectoryURL: URL 주입. 모든 테스트는 FileManager.default.temporaryDirectory.appending(...) 사용. 실제 NSHomeDirectory()는 production 코드 경로에서만 호출.
Preflight 단위테스트가 실제 sshd 의존 MED SSHConnectivityChecker 프로토콜 도입(TEST_STRATEGY §4 L635-642). 단위테스트는 MockSSHChecker로 ok/timeout/refused 케이스 직접 주입. 통합테스트만 실제 ssh 실행.
PairingManager가 Phase 3 PeerDiscovery에 종속 MED PeerChannel 프로토콜로 NWConnection 추상화. PairingManager는 프로토콜만 의존. MockPeerChannel로 단위테스트 전체 커버. 실제 NWConnectionPeerChannel 구현은 Phase 3에서 합류.
FirstLaunchPairingView가 Phase 2 종료 시점에 풀 E2E 페어링 시연 불가 LOW Phase 2는 "peer 대기 화면"까지 출하. 코드 표시 / 수락 UI는 mock-peer 기반 SwiftUI Preview + 단위테스트로 검증. 실기 페어링 시연은 Phase 3 Bonjour 합류 직후.
/usr/bin/ssh-keygen 부재 / PATH 문제 LOW SSHKeyManager init에서 FileManager.fileExists로 검증. 부재 시 명확한 에러 enum (SSHKeyError.toolMissing) 표면화.
Strict concurrency: FSEventStreamCallback C 콜백 → actor 브리지 MED Phase 2 직접 위험은 아니지만 Phase 4(FileWatcher) 사전 학습 필요. ProcessRunner의 NSLock LineBuffer 패턴 재활용 가능.

8. Phase 2 수용 기준 (DoD)

  • PairingCodeGenerator 4개 테스트 통과, 100% 커버리지 (TEST_STRATEGY §8 security-critical 정책)
  • SSHKeyManager: 임시 홈 디렉터리에서 Ed25519 키 생성 → 권한 0600 검증 → install / remove 라운드트립 테스트 통과
  • RemoteLoginPreflight: MockSSHChecker로 ok / connectionTimeout / connectionRefused / hostUnreachable 4가지 케이스 테스트 통과
  • FullDiskAccessChecker: 정상/거부 두 케이스 모킹 가능한 구조 (canary 경로 주입)
  • PairingManager: 정상 흐름 (initiate → code → accept → keyExchange → complete) + reject 분기 + timeout 분기 + double-pair 거부 시나리오 테스트
  • FirstLaunchPairingView: welcome / preflight / code-display 화면 SwiftUI Preview 정상, "Open System Settings" 딥링크가 x-apple.systempreferences: URL 발행 검증
  • Phase 2 신규 테스트가 Phase 1 스모크 4개와 함께 단일 xcodebuild test로 모두 PASS
  • 빌드가 Swift 6 strict concurrency + hardened runtime 설정으로 여전히 BUILD SUCCEEDED
  • HANDOFF.md에 Phase 2 완료 내역 + Phase 3 진입 가이드 업데이트

Q2 · FDA Canary
Full Disk Access 프리플라이트 canary 경로 선택
TECHNICAL_SPEC은 "protected path canary file access test"로만 명시. 후보:
  • (a) ~/Library/Mail/V* 디렉터리 listing — Mail 사용자만 의미 있음
  • (b) ~/Library/Cookies/ listing — 항상 존재, FDA 필요, 권장
  • (c) ~/Library/Application Support/com.apple.TCC/TCC.db 읽기 시도 — 가장 강력하지만 SIP 영역
기본 추천: (b) Cookies. 다른 선택 또는 추가 후보 있으시면 알려주세요.
