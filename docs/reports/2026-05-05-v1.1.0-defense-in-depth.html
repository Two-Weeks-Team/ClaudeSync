4. 단일 인스턴스 + UX fix (사용자 요청 직접 대응)

4.1 Pre-pair UX bug — 사용자 스크린샷에서 발견

v1.0.1 빌드를 직접 실행한 사용자가 보낸 스크린샷에 "Failed: no peer configured" × 5가 누적된 것이 보였습니다. 페어링 전 상태에서 FileWatcher가 file change를 잡고 → SyncCoordinator가 sync job을 enqueue → FileSyncActor가 peer=nil로 즉시 fail → 빨간 X 5개가 사용자에게 노출.

v1.1 fix: FileSyncActor.enqueue가 peer=nil 시 silent drop. setPeer(nil)이 큐 drain. SyncCoordinator가 "no peer" string을 recentResults에서 필터링.

4.2 단일 인스턴스 enforcement

두 인스턴스 동시 실행 시: NWListener bind 실패 / 두 FSEvents 스트림 중복 fire / 동일 PID 영속화 race. SingleInstanceGuard가 두 단계 검증:

  1. NSRunningApplication scan: 같은 bundleId 다른 PID 검색
  2. ~/.claudesync/.app.pid sentinel: 파일 내용을 PID로 읽고 kill(pid, 0)로 liveness 확인

중복 발견 시: 기존 인스턴스 activate → stderr에 명확한 메시지 → exit. graceful shutdown에서 sentinel 정리; SIGKILL 등 비정상 종료의 stale sentinel은 다음 launch에서 자동 정리.
