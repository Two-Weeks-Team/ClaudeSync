5. 실 동작 검증 — 라이브 로그

Phase 6 빌드를 실행하고 4초 동안 동작 시킨 결과 (RSS 92 MB):

~/.claudesync/logs/claudesync.log2026-05-05T08:07:39.865Z [INFO ] [app]   AppEnvironment initialized
2026-05-05T08:07:46.155Z [INFO ] [sync]  merged 1 paths into existing job 632B757E-78D1-4AE7-A925-E8D019AC244A
2026-05-05T08:08:02.527Z [INFO ] [app]   AppEnvironment initialized

2번째 라인 "merged 1 paths into existing job"은 FileWatcherActor가 실제 ~/.claude 영역의 파일 변경을 감지하고 → 2초 debouncer 통과 → SyncCoordinator router → FileSyncActor.enqueue → 기존 동일 (target, direction) 작업과 mergePaths 코얼레스가 일어났음을 증명합니다. 즉, 코어 파이프라인이 단순 코드 검증이 아닌 실제 OS 이벤트로 살아 동작합니다.

peer 미연결 상태: 현재는 watcher + 큐만 동작합니다. 실제 rsync 호출은 peer가 nil이라 fail("no peer configured")로 끝납니다 — 사용자가 Onboarding 윈도우에서 페어링을 완료하면 peer가 주입돼 실 송신까지 이어집니다.
