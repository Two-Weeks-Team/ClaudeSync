5. 핵심 설계 결정

  • PeerChannel 프로토콜 추상화 — PairingManager는 NWConnection을 직접 알지 못함. 실 통신은 NWConnectionPeerChannel이, 단위·E2E 테스트는 LoopbackPeerChannel이 책임. 같은 PairingManager 코드가 두 환경에서 동작.
  • pairConfirm 메시지 추가 — TECHNICAL_SPEC §5에 없는 새로운 케이스. 양측 visual code 확인 후에만 키를 commit하도록 강제. 결과: A가 코드를 보고 거부하면 B는 아무 키도 설치하지 않은 상태로 안전 롤백 (testInitiatorRejectsAfterSeeingCode_responderRollsBack).
  • OSAllocatedUnfairLock — Swift 6.3 SDK에서 NSLock.lock/unlock이 async unavailable 마킹. 채널 내부 상태 락을 os.OSAllocatedUnfairLock withLock 패턴으로 교체.
  • In-process CryptoKit fingerprint — ssh-keygen -lf 셸아웃 대신 SHA256.hash 직접 호출. 테스트 빠르고 결정적, ssh-keygen 부재 환경에서도 검증 가능.
  • homeDirectoryURL 주입 — SSHKeyManager / FullDiskAccessChecker 모두 home 경로 주입 가능. 실제 ~/.ssh/authorized_keys 변조 위험 0.
  • 대칭 상태머신 — PairingManager는 initiator / responder 역할을 메시지 도착 순으로 자동 결정. 단일 코드베이스로 양면 지원.
  • FrameCodec 분리 — NWProtocolFramer는 그대로 두기 어려운 wire format 검증을 pure-Swift StreamReader로 분리. 11개 테스트로 모든 엣지 케이스 (truncated header / oversized / 청크 분할) 커버.
